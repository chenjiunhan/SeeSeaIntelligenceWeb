# SeeSea 🌊

> **用視覺化的方式，理解世界如何運作**

SeeSea 是一個創新的主題式學習平台，透過精美的視覺化和動畫，讓複雜的全球系統變得直觀易懂。我們相信，理解世界運作的最好方式，就是「看見」它。

## 🚢 核心理念

### 主題式探索
- **航運**：我們的第一個主題，揭示全球貿易的脈動
- 未來將擴展至更多主題：能源、金融、供應鏈、氣候等

### 視覺化優先
不只是展示數據，而是**說故事**：
- 🎬 動態動畫展現貨輪在全球航線上的移動
- 📊 實時數據視覺化（港口吞吐量、航運成本、延誤狀況）
- 🗺️ 互動式地圖，讓使用者親身探索航運網絡
- 📈 歷史趨勢與即時數據的對比呈現

### AI 協同學習 ✨
**視覺化 × AI 對話 = 完美學習體驗**

在精美的視覺化旁邊，AI 助手隨時待命：
- 💬 **自然對話探索**：「為什麼這條航線比較慢？」AI 即時分析並在地圖上高亮相關因素
- 🎯 **動態視覺控制**：「顯示過去一週的延誤熱點」→ AI 立即調整視覺化呈現
- 🔍 **智能數據挖掘**：「幫我找出成本效益最高的航線」→ AI 抽取、分析並視覺化結果
- 📊 **即時圖表生成**：「比較這三個港口的效率」→ AI 自動產生互動式對比圖表
- 🎓 **引導式學習**：AI 根據你的興趣，建議下一步探索方向
- 🔄 **持續優化視角**：「換個角度看」→ AI 提供不同維度的數據切片

#### 互動範例
```
使用者：「蘇伊士運河堵塞會造成什麼影響？」
AI：分析歷史數據 →
    1. 地圖上模擬繞行路線（紅色警示）
    2. 顯示延誤天數增長曲線
    3. 標示受影響的航線與船舶數量
    4. 計算額外成本與碳排放
    5. 提供替代方案建議

使用者：「放大看好望角這條替代路線」
AI：視覺化即時調整 →
    - 鏡頭平滑移動至非洲南端
    - 高亮好望角航線
    - 顯示該路線的歷史流量變化
    - 比較繞行成本與時間差異
```

### 數據驅動
結合多元資料來源：
- 實時船舶追蹤數據
- 港口營運資訊
- 航運指數與市場數據
- AI 智能分析與洞察生成

## 🎨 前端設計原則

### 1. 一目了然（Instant Understanding）
使用者進入頁面的第一秒，就能理解「航運是什麼」：
- 主視覺：全球航線動態流動圖
- 核心指標：當下有多少艘船在海上、主要航線、繁忙港口
- 引導式導覽：從宏觀到微觀的探索路徑

### 2. 沉浸式體驗（Immersive Experience）
- 流暢的頁面轉場動畫
- 視差滾動（Parallax）展現不同數據層次
- 互動式元素回饋（hover、click、drag）
- 響應式設計，在任何裝置都有最佳體驗

### 3. 故事化呈現（Storytelling）
不只是儀表板，而是一段旅程：
```
首頁 → 全球航運概覽
  ↓
選擇一艘船 → 追蹤它的航程
  ↓
探索港口 → 了解貨物流動
  ↓
深入分析 → AI 洞察與預測
  ↓
AI 對話 → 「還可以看什麼？」
```

### 4. AI × 視覺化整合設計
**雙欄式布局**：
```
┌─────────────────────────────────┬──────────────────┐
│                                 │                  │
│     主視覺化區域                │   AI 對話區      │
│   (地圖/圖表/動畫)              │                  │
│                                 │  💬 問問題       │
│                                 │  📊 要求視覺化   │
│                                 │  🔍 數據挖掘     │
│                                 │  💡 學習建議     │
│                                 │                  │
└─────────────────────────────────┴──────────────────┘
```

**AI 觸發視覺變化**：
- 對話內容直接控制左側視覺呈現
- AI 回應可包含「視覺化指令」自動渲染新圖表
- 點擊視覺元素，AI 自動提供脈絡解釋
- 語音輸入支援（未來）

### 5. 技術美學
- 簡潔現代的 UI 設計
- 流暢的 60fps 動畫
- 優雅的數據過渡效果
- 深色/淺色主題切換

## 🎯 航運主題展現重點

### 必備視覺化元素

#### 🌍 全球航運地圖
- 3D 地球或平面世界地圖
- 實時船舶位置標記（依船型、載重分類）
- 主要航線流動動畫（如同血管中的血液流動）
- 港口樞紐熱力圖

#### 📦 貨物流動
- 從製造地到消費地的完整旅程
- 貨櫃裝卸動畫
- 供應鏈時間軸

#### 📊 關鍵指標儀表板
- 全球在航船舶數量（實時更新）
- 主要航線繁忙度
- 港口等待時間
- 航運成本指數（BDI, SCFI）
- 碳排放量追蹤

#### 🔍 深度探索功能
- 點選單一船舶 → 查看詳細資訊（航程、貨物、船公司、歷史軌跡）
- 點選港口 → 查看吞吐量、停靠船舶、主要貨物類型
- 時間軸滑桿 → 回放歷史數據、預測未來趨勢
- AI 分析洞察 → 航線優化建議、延誤預警、市場趨勢

## 🎬 使用者旅程範例

### 場景：理解「一個 iPhone 如何從中國運到美國」

1. **進入首頁** → 看到全球航運地圖，數千艘貨輪在移動
2. **AI 引導開始**
   ```
   使用者：「我想知道 iPhone 怎麼運到美國的」
   AI：「讓我帶你看一趟跨太平洋的旅程 📱🚢」
       → 自動聚焦到深圳港，高亮亞洲-北美航線
   ```
3. **跟隨一艘船** → 動畫展示貨輪從深圳港出發
   ```
   使用者：點擊貨輪
   AI：「這是一艘 14,000 TEU 的貨櫃船，載有 8,000 個貨櫃，
        其中約 12% 是電子產品。讓我們跟著它...」
       → 視角鎖定船舶，開始航程動畫
   ```
4. **航程中的互動學習**
   ```
   使用者：「為什麼要走這條曲線路線？」
   AI：分析並視覺化 →
       - 地圖上繪製最短直線路線（灰色虛線）
       - 顯示實際航線（藍色實線）
       - 標註：「順著洋流可節省 15% 燃油」
       - 疊加洋流動畫層
   ```
5. **深入數據挖掘**
   ```
   使用者：「這條航線的成本是多少？」
   AI：即時計算並生成視覺化 →
       - 彈出成本分解圓餅圖
       - 燃油：45%、人力：20%、港口費：25%、其他：10%
       - 「每個貨櫃平均成本 $1,200，iPhone 運費約佔零售價的 1.5%」
   ```
6. **抵達洛杉磯** → 卸貨動畫、清關流程、內陸運輸
   ```
   AI 主動提示：「想看看港口現在有多繁忙嗎？」
   使用者：「好啊」
   AI：切換視角 →
       - 顯示洛杉磯港即時等待卸貨的船舶數量（42 艘）
       - 平均等待時間：3.2 天
       - 對比歷史平均值（1.8 天）→ 生成趨勢圖
   ```
7. **AI 引導延伸學習**
   ```
   AI：「你可能也想知道：
        • 🌍 相同路線的競爭對手航線
        • 📉 運費價格的歷史波動
        • ⚡ 如果走蘇伊士運河會怎樣？
        • 🌱 這趟航程的碳排放量」

   使用者：點擊「碳排放量」
   AI：→ 自動生成碳足跡視覺化，比較海運vs空運vs陸運
   ```

## 🤖 AI 技術架構

### 前端 ↔ AI 通訊協定
```typescript
// 使用者輸入
interface UserQuery {
  type: 'question' | 'command' | 'exploration'
  content: string
  context: {
    currentView: ViewState  // 當前視覺化狀態
    selectedElements: Element[]  // 使用者選中的元素
    conversationHistory: Message[]
  }
}

// AI 回應（可包含多個動作）
interface AIResponse {
  text: string  // 文字回應
  actions: VisualizationAction[]  // 視覺化指令
  suggestions: string[]  // 延伸探索建議
  dataInsights: Insight[]  // 數據洞察
}

// 視覺化動作
interface VisualizationAction {
  type: 'highlight' | 'focus' | 'filter' | 'create_chart' | 'animate' | 'compare'
  target: string  // 目標元素 ID 或查詢條件
  params: Record<string, any>
  duration?: number  // 動畫時長
}
```

### AI 能力模組
1. **自然語言理解**：解析使用者意圖
2. **視覺化指令生成**：將意圖轉換為視覺操作
3. **數據查詢與分析**：從多源數據中抽取答案
4. **圖表自動生成**：根據數據特性選擇最佳視覺化方式
5. **脈絡感知**：記住對話歷史，提供連貫體驗
6. **主動建議**：預測使用者下一步可能的興趣
